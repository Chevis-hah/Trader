# risk/ - 风控模块

## 职责

三层风控防御: 单笔限制 → 持仓止损 → 组合熔断器。

> v2.3 无本目录源码变更；全局说明见 `SYNC_UPDATE_LOG.md`。

## 模块组成

### manager.py - RiskManager

#### 第一层: 事前检查 (`check_order`)

| 检查项 | 默认阈值 | 说明 |
|--------|---------|------|
| 单笔名义价值 | - | 不超过配置限额 |
| 单笔占比 | - | 不超过组合比例限制 |
| 价格偏离 | - | 防止极端价格下单 |
| 订单频率 | 每分钟/每小时上限 | 防止刷单 |
| 持仓集中度 | - | 单标的仓位上限 |

#### 第二层: 持仓级检查 (`check_positions`)

| 检查项 | 默认值 | 说明 |
|--------|--------|------|
| 固定止损 | 3% | 入场价 -3% |
| 跟踪止损 | 5% | 从最高价回落 5% |
| 止盈 | 8% | 入场价 +8% |
| 最大持仓时间 | 168h (7天) | 超时强制平仓 |

#### 第三层: 组合级检查 (`check_portfolio`)

| 检查项 | 默认值 | 说明 |
|--------|--------|------|
| 日亏损 | 3% | 当日 NAV 回撤 |
| 周亏损 | 6% | 当周 NAV 回撤 |
| 最大回撤 | 12% | 全局最大回撤 |
| VaR 监控 | 95%/99% | 实时风险价值 |

#### 熔断器 (`CircuitBreaker`)

**触发条件** (任一):
- 日亏损 > 4%
- 回撤 > 15%
- 连续亏损 >= 5 笔

**触发后**: 冷却 4 小时，期间禁止开新仓

#### 风控快照

`get_risk_snapshot()`: 返回当前 NAV、回撤、日 PnL、敞口、VaR 95/99

## 依赖关系

- **输入**: `execution/position.py` (持仓信息), `data/client.py` (当前价格)
- **输出**: 信号拦截 (阻止违规订单), 熔断状态
- **被调用**: `core/engine.py` 每轮循环调用 check_portfolio + check_positions

## 关键约束

- 风控参数修改直接影响实盘资金安全，修改前必须回测验证
- 熔断器是全局单例，触发后影响所有标的
- VaR 计算依赖历史收益率，数据不足时降级为简单估算
- 止损/止盈检查在 check_positions 中完成，而非策略层
